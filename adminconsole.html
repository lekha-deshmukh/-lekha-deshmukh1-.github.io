Revamping the Analytics Admin App
Before revamp
After revamp

Business Concerns

  • Client Retention: Decrease downtime to boost client satisfaction and retention.

UX Objective

  • Reduce Admin Time: Fewer errors, faster operations.
  • Establish Parity: Unified design across all admin apps to simplify onboarding.

My Role

  • Title: Senior UX Designer (Individual Contributor).
  • Collaboration: Worked with UX Head, CTO, Head of Analytics, PMs, Developers.

Timeline & Tools

  • Duration: 6 months (April – September 2022).
  • Tools:
    • Miro for brainstorming & event-based journey mapping.
    • Adobe XD for high-fidelity prototypes.
Approach & Process

1. Research & Identifying Problems

  • Understanding Current State: Audited existing screens, noted bottlenecks.
  • Comparative Analysis: Studied internal apps for design consistency.
  • Competitor Research: Checked similar analytics platforms.
  • Heuristic Evaluation: Found inconsistencies, siloed information.
  • Event-Based Journey Mapping: Mapped admin journeys to spot friction points.

2. Defining the Problem

  • Top Problems Identified: Prioritized pain points for each screen.
  • Prioritization: Selected issues for immediate fixes.
  • Key Observations:
    • Old UI was just a sitemap with little real context.
    • Crucial information was buried behind multiple clicks.

3. Ideation & Prototyping

  • Summary Dashboard: Widgets surface real-time system health.
  • Widget Design: Focused on key metrics like CPU usage, errors.
  • High-Fidelity Prototypes: Built clickable flows for entire user journeys.

4. Feedback & Improvements

  • User Testing: Admins loved the consolidated dashboard approach.
  • Refinements:
    • Simplified labels for clarity.
    • Enhanced color coding.
    • Added direct drill-down options for troubleshooting.
Expected Impact
  • Early usability tests suggest a 30–35% reduction in troubleshooting time.
  • Simplified dashboard layout brings crucial metrics to the forefront.
  • Fewer navigation steps = faster task completion.
  • Planned metrics:
    • Track ticket resolution times.
    • Measure user satisfaction scores (NPS).
    • Track onboarding efficiency for new admins.
Conclusion & Next Steps
  • Conclusion:
    • Reduced admin time spent managing the app.
    • Unified design across products simplifies training and use.
  • Next Steps:
    • Implement the new dashboard and monitor results.
    • Iterate based on live feedback and real-world performance.
    • Scale the approach to other internal admin apps.
  • Key Takeaway: Shorter admin workflows = higher client satisfaction and loyalty.
